# PRD JSON Setup

## Requirements

No external tools required. Task tracking uses a local JSON file managed directly by the AI.

## Config Variables

| Variable | Default | Description |
|---|---|---|
| `PRD_PATH` | `.toby/{{SPEC_NAME}}.prd.json` | Path to the PRD file for each spec |

Add to `.toby/config.json`:

```json
{
  "templateVars": {
    "PRD_PATH": ".toby/{{SPEC_NAME}}.prd.json"
  }
}
```

## How It Works

- **Plan** reads the spec and writes a structured JSON file with tasks, dependencies, and metadata.
- **Build** reads the JSON, finds the next ready task, implements it, and updates the status in the file.
- The `.toby/*.prd.json` files are the single source of truth for task state.

## File Structure

```
.toby/
  config.json
  01-auth.prd.json      # generated by plan
  02-credits.prd.json   # generated by plan
  status.json
```
